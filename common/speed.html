首屏，当进入该网页并且发送网络请求获取到资源以后，网页从没有内容到内容第一次渲染的时间（当然也可以没有内容）称之为首屏。

结果：优化⾸屏速度可以从两个⽅⾯去优化。

一、网络

当我们网络请求比较多的时候，浏览器会增加⾸屏渲染速度，如果⼤量的数据请求慢，这样会导致⻚⾯渲染也随之较慢，减少⽹络请求

  1. 减少网络请求： 使用图片懒加载形式，只发送用户试图可见的数据请求，这样可以增加请求速度，从而进一步提高渲染速度
  2. 使用缓存： 与后端配合进⾏资源缓存，如⻓期不变的logo或者定量更新的资源可以采⽤缓存的形式来处理，也可以优化⾸屏渲染时间
  3. 使用CDN加速： 对资源进行托管，保证服务器的响应速度

二、加载资源方面

  1. 路由懒加载： 使⽤路由懒加载的形式，进⾏资源处理，当我们需要⽤到该资源时再进⾏资源加载，可以将⼤量的加载时间空出来留给⾸屏渲染。
  2. script标签资源异步加载：当⼀些不影响系统环境资源的加载时，我们可以考虑使⽤ async 和 defer 加载
  3. webpack splitChunks 代码分割：利⽤webpack 代码分割 进⾏优化，可以将 js进⾏分⽚ ，⾸次加载⽂件体积⼤量减少，以及资源异步加载
  4. css 压缩 js压缩 html压缩 image压缩 gzip压缩：
  • css 可以通过 minicssExtracPlugin 进⾏ css压缩
  • js 可以通过 TerserWebpackPlugin | uglifyJsPlugin 进⾏资源压缩
  • html 可以通过 HtmlMinimizerWebpackPlugin 进⾏压缩
  • 图⽚可以通过 ImageMinimizerWebpackPlugin 进⾏压缩
  1. css 样式书写规范： css代码写得好更有利于浏览器的解析
  2. 服务端渲染提⾼⾸屏：服务端渲染 ，后台将你需要绘制的⻚⾯结构以及数据全局准备好了，然后直接将 资源⽂件返回给前 端，前端只需要渲染即可。

补充说明

通过前置和简化模块优化加载速度，主要关注资源的提前加载和模块的简化两方面

  1. 前置关键资源
    • 关键CSS：将关键的CSS样式直接嵌入到HTML中，以避免阻塞渲染。通过内联关键CSS，可以 确保在解析和渲染HTML时，关键样式已经可⽤。这样，⻚⾯的渲染速度会更快，⽤⼾会更早地看到内容。
    • 字体预加载：如果⽹⻚使⽤了⾃定义字体，可以通过预加载字体⽂件来减少字体渲染的延迟。 通过使⽤ <link rel="preload"> 标签，将字体⽂件提前加载，使其在需要时⽴即可⽤。
    • 字体体积瘦身：如果只针对部分文字有特殊字体的需求，可以通过简化缩小字体文件的大小提高加载文件速度。
    • 重要JavaScript⽂件的异步加载：将⾮关键的JavaScript⽂件标记为异步加载，以避免阻塞⻚⾯的渲染。只加载和执⾏必需的JavaScript代码，⽽将其他代码延迟到后续阶段加载。
    • 缓存关键资源：通过适当设置缓存头部相应，将关键资源缓存到⽤⼾的浏览器中。这样，⽤⼾在后续访问⽹站时可以直接使⽤缓存的资源，减少⽹络请求的数量和响应时间。
  2. 简化模块
    • 代码切割：将代码分割成较⼩的模块（chunks），按需加载。通过将⻚⾯的不同部分拆分为独 ⽴的模块，可以仅在需要时加载这些模块，⽽不是⼀次性加载整个应⽤程序的代码。这样可以 减少⾸屏加载时需要下载的代码量，加快⻚⾯的加载速度。
    • 按需加载：延迟加载⾮关键的模块或组件，只在⽤⼾需要时进⾏加载。例如，将某些交互性较低的模块或组件延迟加载，以减少初始加载时的资源消耗。
    • 优化第三⽅库和插件：检查使⽤的第三⽅库和插件，确保只加载和使⽤必需的部分。有时候，第三⽅库可能包含⼤量不必要的功能和代码，可以通过按需导⼊或选择性地加载来减少⽂件⼤⼩和加载时间。
    • 压缩和缩减⽂件⼤⼩：通过压缩和缩减⽂件⼤⼩来减少模块和资源的加载时间。可以使⽤⼯具如Webpack或Parcel来进⾏代码压缩、⽂件合并和优化。
